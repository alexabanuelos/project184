Project: Texture The Tamagotchi!


Patrick Sanchez, Karson Du, Ryssa Malibiran, Alexa Banuelos

Summary

Currently we have implemented a simple scene in WebGL through three.js. Inside we have some examples of built-in custom materials and 3D primitive meshes, interactive camera functionality, and a simple light source. In addition, we added a custom texture onto a distorted plane using a mesh we defined manually, with a texture we imported. With blender we also constructed a shell of our Tamagatchi game device, exported in gltf (as desired by three.js).

Preliminary Results

From the beginning there was some debate around where we should start. For example, we discussed taking code from Homework 2 that already had shading built in, and would allow us to possibly upsample our shell's mesh, but found it was unrealistic to work through the staff skeleton's code without guidance. The other homeworks had the same concern with the barrier of entry, especially since Homework 3's rendering would not be efficient enough for us to get to implement the game portion of our project proposal. Homework 4's simulation had some more hope, but it already had a staff texture handler built in using GLSL. Working off of the homeworks wouldn't noticeably accelerate our progress towards recreating the Tamagotchi environment we wanted.

While we were deciding if we should pivot our project idea slightly to fit the proposal, we thought of previous projects adjacent to our idea. One of our favorites built their projection model using three.js, which is also responsible for some popular games we had played before (Crossy Road, Krunker.io), which surprised us! Since .gltf 3D models include UV mesh information, we’re taking our current texture test as a proof of concept for implementing texture maps with standard volume projections. Since shapes such as boxes, spheres, and cylinders all have some merit for possible textures, we’d like to configure the code to have a UI that enables users to visualize textures using these shapes, and choose which one to apply to the tamagotchi skin. This felt like a good compromise, since we have already implemented Blinn-Phong shading as well as texture filtering methods in previous homeworks. Thus, spending more time on interaction with visuals and the in-between steps of projection over these features gives us new content to cover.

Reflection: Goals and Deliverables

With these new ideas in progress, depending on how implementing planar, cubic, spherical, and cylindrical coordinate mapping goes, we hope to finish the visualizations not too long after. Given two weeks, optimistically we can also find out how to take user inputs for uploaded custom images as textures, and retain our plans for drawing textures live inside of the website. From there, we can reassess gameplay with whatever of the timeline we have remaining!
